Notes: Using WebKB

This is a supplementary documentation for WebKB, targeted on programmers wishing to exploit one of the ontology handling features of WebKB.

What is WebKB

WebKB is a web accessible Knowledge Base that contains a type hierarchy + the knowledge statements.

We are mainly interested in the taxonomy. An Ontology consists of category types interconnected by links. For purpose of this document we will call them ' relation links '.

Table of relation links and corresponding symbols:

Some other related terminology

Name - a word describing an entity you wish to search for, e.g. 'cat', 'dog'. ( Case sensitive )

Identifier - unique name given by WebKB to identify the category, e.g. 'wn#cat' is identifier for 'cat'.

Identifier Format: 'prefix#word', where:

  • prefix is a creator name (e.g. 'pm') or domain name (e.g. 'wn' for WordNet or 'comms' for telecommunications ontology). In 'wn' case prefix can be omitted.
  • word is an english word. Most likely users will be using term names in their searches, and programs will be using identifiers. One word can have a more then one meaning and each meaning has unique identifier. For example: cat can have following meanings: #cat, #true_cat, #big_cat, #Catterpillar, etc

WebKB RDF Output Format

To start with we used RDF output format produced by WebKB. We are using RDF parser that can process RDF file and produce a set of triples. As a result, we get a set of statements such as: Subject->Predicate->Object. In our case each Subject will correspond to an ontology node/type/term, Object can correspond to ontology node/type/term as well as to a literal string. Predicates describe relation links between ontology nodes. So, after processing a statement, we may get an ontology node connected to another ontology node by some relation link. Or we may get ontology node connected to a literal with some relation link.

Here is table of relation links to RDF Properties mapping.

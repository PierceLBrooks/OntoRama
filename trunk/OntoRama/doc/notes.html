OntoRama Notes

Configuration File config.xml 

Location: by default is located in 'classes' directory

Elements 'ontConfig/ontology/relation' and 'ontConfig/otology/relation/relationType'

Describe all available relation links, assign each of them an id number. Each relation link should have a name. Reversable relation links also have a second name.  

For example: relation link subtype/supertype is assigned id="1". It's name is 'subtype'.
 So, if 'wn#feline' is a parent of 'wn#cat', we will have relation link with id="1" , name of relation link is 'subtype', relation is from 'wn#feline' to 'wn#cat'. 
If we were to reverse this relation and wanted to talk about supertypes, we would then use name 'supertype' and reverse order of 'wn#feline' and 'wn#cat': now we have relation link id="1" and relation link 'supertype', relation is from 'wn#cat' to 'wn#feline'.

NOTE: make sure attributes 'id' are in order from 1 to n. For example: 1,2,3,4 is valid; 1,2,4,5 is not valid.

 

Element 'ontConfig/ontology/relation/display'

Describes display properties for this relation: colour, icon.

If icon is not specified, OntoRama will create an icon with given display color and mappingSymbol.

Element 'ontConfig/ontology/conceptProperty' 

Attribute 'id' - is a description of conceptProperty that will appear on Description Panel.

Element 'ontConfig/ontology/rdfMapping'

NOTE: map id's should correspond to id's in elements 'relaition'. Otherwise - you will get an error while parsing RDF about unknown property type. Check config.xml file first before debugging RDF parser ;)

<ontConfig>

    <ontology>

        <relation id="1">

            <relationType name="subtype" mappingSymbol="&lt;"/>

            <relationType name="supertype" mappingSymbol="&gt;"/>

        <display color="#0000ff" icon="someIconPath"/>

    </relation>

        ......

    <conceptProperty id="description"/>

    <conceptProperty id="creator"/>

    </ontology>

<rdfMapping>

    <map id="1" type="supertype" tag="subClassOf"/>

    <map id="2" type="synonym" tag="label"/>

    <map id="4" type="part" tag="part"/>

    <map id="7" type="inclusive" tag="jointWith"/>

    <map id="description" tag="comment"/>

    <map id="creator" tag="Creator"/>

 </rdfMapping>

</ontConfig>

Configuration File 'ontorama.properties'

Location:    classes dir

This file is read by the application to set basic properties:

  • sourceUri - URI of ontology source. Can be URL of some web accessable file, for example: http://meganesia.int.gu.edu.au/~nataliya/OntoRama/test/web-examples/imdb.rdf. Or, can be file on the local machine. In this case location of the file should be formulated as an URI, for example: file:/E:/projects/OntoRama/test/metanet.rdf
  • ontologyRoot - root of ontology described in sourceUri. For example, for Webkb ontology wn_cat.rdf, root should be 'wn#Cat'.
  • queryOutputFormat - expected format of data that OntoRama will receive. In some cases, this parameter may also act as an parameter for an API that indicates that OntoRama expecting this certain output format. Presently, we only support RDF and XML.
  • parserPackagePathSuffix - specify a package that can parse data OntoRama is receiving from sourceUri. For example, we wrote RdfDamlParser for queryOutputFormat=RDF and XmlParserFull for queryOutputFormat=XML. However, there may be a need for other parsers for different XML or RDF schemas or possibly for different input formats (EBNF for example). This parameter allows us to switch between parsers without rebuilding the application.

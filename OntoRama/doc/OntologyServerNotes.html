Notes: Using WebKB

This is a supplementary documentation for WebKB, targeted on  programmers wishing to exploit WebKB capabilities.

What is WebKB

WebKB is a web accessible Knowledge Base that contains a type hierarchy + some knowledge statements.

We are mainly interested in the taxonomy. Ontology consist of category types  interconnected by links. Interconnecting links are called 'relation links'.

Some other related terminology

Name - a  word describing an entity you wish to search for, e.g. 'cat', 'dog'.  ( Case sensitive )

Identifier - unique name given by WebKB to identify the name you are searching for, e.g. 'wn#cat' is identifier for 'cat'.

Identifier Format: 'prefix#word', where :

  • prefix is a creator name (a.e. 'pm') or domain name (a.e 'wn' for WordNet or 'comms' for  telecommunications ontology). In 'wn' case prefix can be omitted.
  • word is an english word. Most likely individuals will be using term names in their searches and programms will be using identifiers (according to WebKB documentation).  One name can have a more then one meaning and each meaning can have unique identifier. for example: cat can have following meanings: #cat, #true_cat, #big_cat, #Catterpillar, etc

 What we can do with WebKB  Ontology

In order to avoid to specify full WebKB URL's we will assume that the base URL is known and use relative paths. For instance, currently WebKB is installed at following location: http://meganesia.int.gu.edu.au/~phmartin/WebKB

Main functionality we are interested in is following:

  • search
  • compare
  • modify

Searching 

Search  for types and relationships between types.

CGI path:    bin/termSearch.cgi

CGI Parameters

Parameter Name Expected Parameter Value Description Required
term Category name or Identifier Term you are searching for yes
link Relation link between terms   no
destTerm Category Identifier Used to find direct connection between two terms. 
Terms in question are specified by parameters 'term' and 'destTerm' and connection is specified by 'link'. 		 no
recursLink Relation link Used to set constrains on the search for a given term.
If this parameter is not set - WebKB will return search term and all terms connected to it via direct relations.
Otherwise recursLink can be set to any of valid relation links. This will return the above set of data plus all terms connected to search term via recursLink.
However, only a limited section of a large search will be displayed		 no
depth an integer Use in conjunction with recursLink to specify the depth of recursion you require. no
creator user identifier Used to search for terms created by certain person or of a certain domain. no
nonCreator user identifier The inverse of 'creator' parameter.  no
format FCG
RDF		 Used to specify format of output data.
Result available in following formats:
- if none is specified - default is 'minimal, user friendly'
- FCG - EBNF format wrapped in html between <pre> and </pre> tags
- RDF 		 default value is
 'minimal, user friendly'
hyperlinks   This parameter is a flag that indicates if returned terms should be hyperlinked.
If this parameter is present - terms are hyperlinked		 no
